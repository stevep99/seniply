Seniply

Download 

Seniply is an ergonomic, minimal keyboard layout for a keyboard with at least 34 keys. The minimum requirement is 30 keys in the main body plus 4 thumb-keys. All the features (and more) of a standard full-size keyboard are available by making use of six layers, which are activated via the thumb keys. The default base layer is Colemak-DH.

It is designed to be compatible with both small ergo keyboards and traditional keyboards. Special firmware such as QMK isn't required and non-programmable boards are well-supported: the reference implementation is achieved using KMonad.

See the hardware section below for more details on recommended keyboard types.

❮ ❯

Features
  • Low travel distance by keeping fingers near their home positions; movements larger that 1 key-unit are avoided.
  • Well positioned thumb-keys are used to access layers - six layers are supported - with no need for dual-role keys.
  • Sticky (one-shot) modifiers are used to fully support Ctrl, Alt, etc, without the need for dedicated keys. No more modifiers in awkward corners of the keyboard.
  • Layer mappings designed to be intuitive and easy-to-learn.
  • Easily adjustable and expandable to meet user requirements (e.g. to support multilingual typing).
  • Compatible with standard staggered keyboards, but for best results, a keyboard with a split spacebar is recommended.

Layers

Base Layer

The base alpha layer is Colemak-DH, modified slightly so that apostrophe is moved to the semicolon location. Being the more frequent character, apostrophe is more deserving on a spot on the base layer when using a minimalist layout. (Semicolon and colon are still easy to access on a symbols layer). The second layer is the standard Shift layer, and is selected by holding down the inner-left thumb-key, located on the bottom-most row, approximately below the T key. The space bar is by default on the right thumb.

Extend Layer

Navigation and other non-printable functions are provided by the Extend layer, which is heavily based on DreymaR's original. The Extend layer is selected via the left-most thumb key, typically located below the S key. This layer provides navigation, page up/down, home/end on the right hand side, and backspace is assigned to a comfortable spot on a strong index finger (Ext-H).

On the left-hand side are home-row modifiers for Alt, Super, Shift, Control and AltGr on A,R,S,T,G respectively. As with DreymaR's Extend, these modifiers can be held down and combined with other Extend functions such as arrows etc, to comfortably produce combinations like Ctrl ⌫, Alt ←. Importantly though, these home-row modifiers are implemented as one-shot (sticky) keys, which makes it easy to do Control and Alt shortcuts without needing dedicated modifier keys.

Examples:
(i) Output Ctrl-A using Ext-t a
(ii) Output Ctrl-W using Ext-t w
(iii) Output Alt-Tab using Ext-a-d

Function Layer

To accomodate the reduced number of keys, the function keys are mapped in a dedicated Function layer. The function keys are on the right-hand side, following a familiar Numpad-like convention for ease of learning. As with the Extend layer, home-row modifiers are available to combine with function keys. The function layer also provides a mechanism to switch between layouts (e.g. to switch to Qwerty for gaming) via the @L1,@L2,@L3 mappings. To access the function layer, the two outmost thumb-keys are held down. Alternatively, on keyboards with good support for multiple thumb keys, you may assign an unused thumb key for selecting this layer.

Examples:
(i) Output F11 using Ext-Sym-m
(ii) Output Ctrl-F5 using Ext-Sym-t-e
(iii) Output Alt-F2 using Ext-Sym-a-,

Symbols Layer

US variant    UK variant

Additional symbols are provided by Symbols1 and Symbols2 layers. The Symbols1 layer has the most frequent symbols and is activated by holding the right-most thumb key, usually located beneath E. Numbers and basic mathematical symbols on the right-hand side, arranged in NumPad style for convenience. The most common remaining symbols, such as brackets, colon and semicolon, and the shifted number characters are on the left-hand side. For easy recall the top-row symbols on the QWFPB keys match the shifted-number symbols for 12345. Note also that each Bracket type is assigned a finger, e.g. ( and ) are both on the index finger - this does not cause a same-finger bigram since open and closing brackets are not usually typed together, and in fact the opening bracket is more common as many editing tools (such as IDEs) automatically fill in the closing bracket, hence in this layer it is the opening brackets that occupy the home positions.

US variant    UK variant

The Symbols2 layer has the remaining standard symbols found on keyboards, such as ~ \ |. It also makes available some mathematical symbols not usually found in most layouts, such as ½ … ± ÷ ×, but these can be customized if you have different characters you are more likely to type. When using the KMonad implementation, for these characters to be printed correctly, you must ensure compose key combinations are configured consistently in both KMonad and the OS. On systems that don't support compose characters natively, this feature can be provided by an external tool, such as WinCompose for Windows.

A couple of dead-keys have also been added, labelled d' and d¨ for producing accented characters such as é and ö. The Symbols2 layer is activiated by first holding the Symbols1 thumb key and then holding down Shift. On keyboards with good thumb key support you could opt to use a dedicated single key instead.

One further note regarding symbols, it is possible to type any of the symbols that would normally be made available via the AltGr key, where configured. For example, if AltGr-a usually produces æ on your system, you can reproduce this with Ext-G a. Using this technique can substantially increase the number of symbols that are available for you to type using keys right under your fingertips!

Hardware

Seniply works best on the following types of hardware:

  • Small ergonomic boards such as the Planck, Kyria, Atreus, Corne, Lets Split, etc.
  • Larger ergonomic boards such as the Preonic, Ergodox, Moonlander, etc. In this case, usage of the additional keys is optional.
  • Traditional staggered keyboards that are physically split or have split spacebar, such as Ergo Pro, UHK, Dygma Raise, Mistel Barocco, or similar.

It is also works satisfactorily on traditional staggered keyboards with smaller (at most 5u wide) spacebar, such as is typically found on laptops. On these keyboards the Wide Mod is required.

Other keyboards, especially those with a large spacebars, are not suitable for Seniply's thumb-based layer selection system. But you may adapt it somewhat by using other keys, such as CapsLock and Enter, as viable alternative layer selection keys.
